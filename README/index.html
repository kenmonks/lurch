Lurch.plus — Eleventy Source Conventions

This folder contains the Eleventy source for https://lurch.plus.
Everything here exists to build the static site; nothing here is deployed directly.

Core ideas

  • Markdown-first content
  • YAML for configuration/data whenever possible
  • Minimal Eleventy magic, but consistent use of its data cascade
  • Site styling and math rendering match the Lurch editor exactly

Folder structure

  • data/

    • Global site data (YAML/JS)
    • Examples: site.yml, nav.yml
    • Available in templates as site, nav, etc. (namespaces)

  • includes/

    • layouts/ — Liquid layouts (pages)
    • partials/ — Reusable UI fragments (menus, headers, etc.)

  • assets/

    • Static assets copied through as-is
    • CSS, JS, images, icons

  • Content folders (e.g. learn/, theory/)

    • Markdown source files
    • Optional per-folder defaults via *.meta.yml or *.yml where * is the folder name.

Directory defaults (*.meta.yml or *.yml)

Folders may contain a YAML file named after the folder, with suffix .meta.yml. or .yml

Example: learn/proofs/proofs.yml or learn/proofs/proofs.meta.yml

These files define defaults like:

  • layout
  • navMode
  • courseId

They apply to all Markdown files in that folder and below.

Layouts

Layouts live in includes/layouts/.

Common layouts:

  • lurchplus.liquid — project pages
  • course.liquid — course / textbook content

Layouts are applied via directory defaults or front matter.

Navigation

Navigation is hand-rolled, not Eleventy TOC-based.

  • Menu rendering: includes/partials/menu.liquid
  • Behavior (collapse/expand): assets/js/utils.js
  • Structure defined in data/nav.yml
  • Pages select a navigation mode via navMode

Math

  • Math rendered with MathJax
  • Custom LaTeX macros defined globally
  • Styling matches Lurch editor contexts exactly

Documentation-only files

Files like this README exist for developers only and are excluded from output using:

eleventyExcludeFromCollections: true

Navigation & Collections (Eleventy Quick Reference)

Eleventy Navigation Plugin — What it is

Eleventy’s navigation plugin builds a tree structure of pages based on metadata you add to front matter. That front matter describes the ancestor-dependency tree.

Minimal front matter example

eleventyNavigation:
  key: "Intro to Proof"

Nested example:

eleventyNavigation:
  key: "Chapter 1"
  parent: "Intro to Proof"
  order: 1

Eleventy collections — What they are

A collection is just an array of pages with metadata. Each item represents one rendered page and includes: • url • inputPath • outputPath • merged data (front matter + folder + global)

Built-in collection • collections.all → all renderable pages (most common input for navs, indexes, sitemaps)

Custom collections (in .eleventy.js)

eleventyConfig.addCollection("chapters", api =>
  api.getFilteredByGlob("./src/learn/proofs/ch*.md")
)

My Custom Markdown classes

Using the markdown-it-div and markdown-it-attr plugins allows me to define custom markdown wrappers and attributes via css.

Mult-column Blocks (equally spaced)

Usage:
::: cols cols-2 middle gap-md stack-sm
  ::: col
    
  :::   
  ::: col
  
  :::
:::

Note that markdown allows fenced blocks to be indented up to 3 spaces for nesting (the indentation of the fence is removed from the indentation of the content as well)

Options:

  • cols-2, cols-3, cols-4
  • gap-sm, gap-md, gap-lg
  • left, center, right
  • top, middle, bottom

Side-by-side panes

Usage:
::: split left-25 
  ::: pane center middle
    
  :::   
  ::: pane left
  
  :::
:::

Note that markdown allows fenced blocks to be indented up to 3 spaces for nesting (the indentation of the fence is removed from the indentation of the content as well)

Options:

  • split
    • left-20, 25, 33, 40, 50, 60, 66, 75, 80 (thirds, fourths, fifths)
  • pane
    • gap-sm, gap-md, gap-lg
    • left, center, right
    • top, middle, bottom
